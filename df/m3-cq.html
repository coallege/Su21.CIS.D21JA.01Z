Why can't memory operations be implemented with 2 steps instead of 4?

From the Assembly Language for x86 textbook, it says:

"2.1.3 Reading from Memory
...reading a single value from memory involves four separate steps:
1. Place the address of the value you want to read on the address bus.
2. Assert (change the value of) the processorâ€™s RD (read) pin.
3. Wait one clock cycle for the memory chips to respond.
4. Copy the data from the data bus into the destination operand."

QUESTIONS:

  1. Why can't operations 1 and 2 be combined (place address value on bus and set RD pin simultaneously)?
  2. Why can't operation 3 write data directly into the destination operand (REG/ALU) to eliminate step 4)?

I understand these are CPU hardware questions that don't really relate to ASM software.  But this section is on CPU architecture and I'm curious why there needs to be 4 steps for the hardware to read memory.

fixed frog posted at 2021-07-06T21:54:02.000Z

I believe this is an issue with hardware limitations and how the memory is made to function to prevent overflow and corruption. Not 100% sure, but A can't happen physically because memory addresses and memory storage are two different things. They are not the same. The machine loads the "address", finds "address", and then goes to this address. B is due to how memory is handled, in this instance all memory is stack-based. Once the memory is loaded into the stack, it needs to check if it can go off the stack. Different memory systems have different limitations. For instance, a hard drive only spins one way so it needs to cycle again to go back in memory.

spatial snipe posted at 2021-07-12T02:15:37.000Z

This is certainly the limitation of Hardware. Since electrical signals and clock controls the processes, you can do only one action per clock cycle, read or store or add.  All these low level instructions on how to read memory or how to store memory gives immense power to write the programs efficiently at the same time, it can be super lengthy.  However as we move up to higher level languages, all these low level actions are abstracted and we never need to worry about those details, but at the cost we need to give into limitations of high-level languages like C, C++, Java etc.   For your query about combining steps, it may be possible with redesigned hardware processors, but I am not sure if they will be feasible enough, otherwise all these improvements might have been done since the processor industry has come a long way.

faithful firefly posted at 2021-07-12T04:03:12.000Z
If you combined the step of changing the read pin and writing out the hardware address, you might have a race condition where the read pin is changed before the address is changed, and the memory would fetch the wrong address. Because the smallest unit of machine time is the clock cycle, it might be infeasible to subdivide a single clock cycle into waiting for an address writing-out state and then completing a read pin changing state. Besides, as is mentioned in the book a lot of this lengthiness is mitigated by the fact that caching exists.

Purpose of Sign Status Flag

In the previous chapter, we learned that in 2's complement, the sign bit is used to indicate sign. Then, what is the purpose of the sign status flag? Is it just for convenience?

driving deer posted at 2021-07-12T03:02:27.000Z

Hi Kavan,

I believe that the sign bit, which is used to indicate whether a 2's complement number is positive or negative, from a resulting mathematical operation helps set the sign status flag. If the sign bit of the resulting number is 1, then the sign status flag is set to 1. Contrastingly, if the sign bit is 0 and the resulting number is positive, then the sign status flag is set to 0. Since the bit for the flag is stored inside a register in the CPU, I believe that it is important for the processor when executing instructions. Wikipedia has a nice explanation of the sign flag:https://en.wikipedia.org/wiki/Negative_flag (Links to an external site.)
