Very basic question:

PUSH ebx -> save register values

POP ebx -> restore register values

One of the homework assignments requires (Preserve all registers (except EAX) that are modified by the procedure).

Preserve in this context means PUSH? The more I read it the more I think it is POP. Confused.

thoughtless swordtail

When they say preserve registers, they mean to push the registers you will modify at the beginning of the procedure (after you have created your stack frame) and pop the registers you modify at the end of the procedure (before you leave the stack frame). You can also use the USES keyword to preserve the registers which will do the same thing as I mentioned, however, I doubt we will be able to use it in the final exam. Also, the homework asks you to use MASM parameters and I think pushing and popping registers manually doesn't play well with the MASM parameters and you should use USES.

forward frog

To clarify the usage of push and pop in a procedure.

mov eax, 10

Test1 PROC

Push eax ; saves the value of eax =10  when modified.

add eax, 10

mov ecx, eax ; eax is added to loop counter

pop eax ; eax is now 10 since new eax is poped and not used.

In this case eax is preserved its value outside of the PROC
